The technical audit examines 26 elements across crawlability, indexation, site speed, security, and server-level configuration — the infrastructure search engines depend on to access and rank your content.

Three areas require immediate attention: 47 broken URLs are wasting crawl budget and losing link equity; mobile page speed scores in the "Poor" range at 31/100; and roughly 30% of the site's pages remain unindexed. Addressing these three items will have the most direct impact on organic visibility.

4xx Errors & Broken Links

expand_more

What it is

Client-side errors where the requested page cannot be found or accessed by search engines.

Why it is a problem

It wastes crawl budget and creates a "dead-end" experience for users and bots alike.

How to fix it

Redirect broken URLs to relevant live pages or update internal links to point to active content.

speed

Page Speed — Mobile

expand_more

What it is

Mobile page speed measures how quickly the site loads on mobile devices via Google's Core Web Vitals framework. It directly affects both search rankings and the percentage of visitors who stay on the site.

Why it is a problem

The mobile PageSpeed score is 31/100, placing the site in the "Poor" range. Largest Contentful Paint averages 6.2 seconds — more than double the recommended 2.5 second threshold. The primary cause is uncompressed images and render-blocking scripts.

How to fix it

Convert hero images to WebP format, implement lazy loading, and defer third-party scripts that are not critical to initial render. Enable server-side caching and consider a CDN for static assets.

search_check

Indexation Coverage

expand_more

What it is

Indexation coverage refers to the proportion of the site's pages that are successfully indexed by Google and eligible to appear in search results.

Why it is a problem

Search Console shows 214 pages in a "Discovered — currently not indexed" state and a further 38 marked "Crawled — currently not indexed." Combined, these represent roughly 30% of the site's total page count.

How to fix it

Audit non-indexed pages to separate noindex-blocked pages from thin-content pages. Apply canonical tags on faceted navigation pages. Add unique introductory content to thin category pages with commercial intent.

account_tree

XML Sitemap

expand_more

What it is

The XML sitemap is a structured file that tells search engines which pages should be crawled and indexed. A well-maintained sitemap helps ensure important pages are discovered promptly after updates.

Why it is a problem

The sitemap includes 83 URLs that return a 301 redirect and 12 URLs currently marked noindex. Including redirected or non-indexable URLs creates conflicting signals and wastes crawl budget.

How to fix it

Remove all redirecting URLs from the sitemap — only final destination URLs should be included. Remove all noindex URLs, and update CMS sitemap generation settings to dynamically exclude non-indexable pages.

link_off

Redirect Chains

expand_more

What it is

Redirect chains occur when a URL redirects to another URL that itself redirects again before reaching the final destination. Each hop dilutes link authority and adds latency.

Why it is a problem

The crawl identified 31 redirect chains, some reaching 3 or 4 hops. These involve legacy product URLs redirected across two separate site migrations and never consolidated. Each extra hop passes progressively less authority to the destination.

How to fix it

Update the origin URL's redirect rule to bypass intermediate URLs and resolve in a single hop. Refer to the Redirect Plan report for the full list of chains to consolidate.

language

Hreflang Configuration

expand_more

What it is

Hreflang tags tell search engines which language and regional version of a page to serve to users in different countries.

Why it is a problem

The French-Canadian and English-Canadian versions are missing reciprocal hreflang annotations on a subset of product pages. Google may surface the wrong language variant to users.

How to fix it

Ensure every page with an hreflang tag also includes the reciprocal tag. Add a self-referencing hreflang tag on every page. Validate with a hreflang testing tool after deploying.

URL Structure

expand_more

What it is

URL structure refers to how page addresses are formatted — including the use of hyphens, lowercase characters, logical hierarchy, and avoidance of unnecessary parameters.

Why it is a problem

URL structure is well-implemented across the site. All URLs use lowercase characters, hyphens as word separators, and a logical folder hierarchy. No action is required at this time.

How to fix it

Maintain the existing URL conventions when adding new pages or categories. Avoid introducing dynamic parameters for filterable content without canonical tags.
